# Upload File to URL — Workflow Action

**Upload File to URL** sends a file to a pre-authorised upload URL. It is the second half of Scram's two-step file upload pattern — it takes a file from the user and a URL prepared by the server, and performs the actual transfer.

---

## What is Upload File to URL?

File uploads in Scram follow a **prepare-then-upload** pattern. Rather than sending a file directly through your own backend (which would require your server to handle potentially large binary data), Scram uses a pre-signed URL approach:

1. A backend action (**Prepare File Upload**) asks the file storage service to generate a short-lived, pre-authorised upload URL
2. **Upload File to URL** sends the file directly from the user's browser to that URL

This keeps large file transfers out of your application logic entirely — the file goes straight from the user's device to the storage service.

Upload File to URL handles the actual HTTP transfer, progress, and success/failure reporting. You do not need to write any file-handling code.

---

## Inputs

### File

The file to upload. This must be a **File type value** — typically sourced from a file input component's exposed variable (e.g. `components.fileInput.vars.value`).

It cannot be a URL, a base64 string, or any other file representation. It must be the raw File object as captured from the user's input.

### Upload to URL

The pre-signed upload URL generated by a preceding **Prepare File Upload** step. This is taken directly from that step's output:

```
steps.prepareUpload.output.value.uploadUrl
```

This URL is short-lived and single-use — it is generated specifically for this file and this upload. Do not reuse URLs across multiple uploads or store them for later use.

---

## The Full Upload Pattern

Upload File to URL is never used in isolation. The complete pattern is:

```
1. Prepare File Upload
   — Calls the file storage service to generate an upload URL
   — Returns: uploadUrl, and the permanent file URL for storing in the database

2. If/Else (did Prepare succeed?)
   true:
     3. Upload File to URL
        — file:      components.fileInput.vars.value
        — uploadUrl: steps.prepareUpload.output.value.uploadUrl

     4. If/Else (did Upload succeed?)
        true:
          5. ExecuteSQL — store steps.prepareUpload.output.value.fileUrl in the database
          6. SetVariable vars.uploadedFileUrl = steps.prepareUpload.output.value.fileUrl
        false:
          5. SetVariable vars.errorMessage = "Upload failed. Please try again."

   false:
     3. SetVariable vars.errorMessage = "Could not prepare upload. Please try again."
```

The permanent file URL — the one you store in the database and display in the UI — comes from the **Prepare File Upload** step, not from Upload File to URL. It is known before the upload happens because the storage location is determined when the upload URL is generated.

---

## Output

After a successful upload, `steps.stepId.output.success` is `true`. The step itself does not return a file URL — that comes from the Prepare File Upload step.

On failure, `steps.stepId.output.success` is `false` and `steps.stepId.output.errorCode` indicates what went wrong.

Always follow Upload File to URL with an **If/Else** before proceeding to store the file URL or update the UI.

---

## Common Failure Cases

| **Scenario** | **What happens** |
| --- | --- |
| The upload URL has expired | The upload fails with an auth/forbidden error |
| The file is too large for the storage service's limit | The upload fails with a size error |
| The user's network connection drops mid-upload | The upload fails; the database is not updated |
| Prepare File Upload failed and the URL was never generated | Upload File to URL should never be reached — caught by the first If/Else |

Because the database write only happens after a confirmed successful upload (in the true branch of the second If/Else), a failed upload leaves no orphaned records pointing to a file that does not exist.

---

## Important Constraints

- **File input only** — the `file` input must be a File object from a file input component. No URLs, no base64.
- **One file at a time** — each Upload File to URL step handles a single file. For multiple files, the pattern repeats for each one.
- **URLs are single-use** — the pre-signed URL from Prepare File Upload is for one upload only. If the upload fails and the user retries, a new Prepare File Upload step must run to generate a fresh URL.
- **Frontend only** — this action runs in the browser, not on the server. The file transfer happens directly from the user's device.

---

In summary: Upload File to URL is the **transfer step** in Scram's file upload pattern — it does the actual work of sending a file to storage, but only once Prepare File Upload has laid the groundwork. The two actions always work as a pair, with If/Else checks at each stage to ensure a failed transfer never results in a broken or missing file reference in the database.
