# Validate Password

# Validate Password — Workflow Action

**Validate Password** checks a password against a user's stored password. Its main use is as a gate before Set Password, to confirm a signed-in user knows their current password before letting them change it.

---

## What is Validate Password?

It answers one question: does this password match what is stored for this user? It does not change anything — it only checks. You use its result to decide whether to proceed, typically to a Set Password step.

---

## Inputs

| **Input** | **Type** | **Notes** |
| --- | --- | --- |
| **userId** | text | The id of the user to check against. Takes the user id, not the email — resolve the user to an id first. This is natural when the user is already signed in, since the id is on the session. |
| **password** | text | The password to check. |

---

## Returns

The result of the check — whether the supplied password matched the stored one. Branch on it to decide whether to continue.

---

## How it Works at Runtime

1. It looks up the user by `userId`.
2. It compares the supplied password against the stored password.
3. It returns whether they matched. Nothing is written.

---

## Common Patterns

### Guarding a password change

```text
Step 1: Validate Password(userId, currentPassword)
Step 2: branch on result
          → matched:   Set Password(userId, newPassword)
          → no match:  show "current password incorrect"
```

The signed-in user must prove they know their current password before it is replaced.

---

## Important Constraints

- **Read-only.** It checks; it never changes a password. Pair it with Set Password to act on the result.
- **Identified by userId, not email.** Resolve the user to an id first — straightforward when they are signed in.

---

## Relationship to Other Actions

- **Set Password** — the action you gate with Validate Password when a current-password check is required.

See [recipe 5](/recipes/auth-flow-recipes#5-change-your-own-password-signed-in) for the change-own-password flow.

---

In summary: Validate Password is the **check** step for a current password. It reads and returns a match result; combine it with Set Password when you need to confirm the old password before setting a new one.
