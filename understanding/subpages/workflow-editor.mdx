The Workflow Editor is a separate canvas that opens when you edit a workflow. It replaces the main Canvas while you're working on logic, and you return to the page via the **Back** button in the top left.

---

## Layout

The Workflow Editor is a freeform canvas centred around the workflow's trigger — the event that starts the workflow, such as **On click**. Actions are added below the trigger and flow downwards, forming a visual chain.

A **minimap** in the bottom right corner gives you an overview of the full workflow, useful for navigating longer or more complex chains.

---

## Adding actions

Click the **+** button below any step to open the **Add Action** modal. This lets you choose the next action in the workflow — such as Execute SQL, Set Variable, IfElse, and so on.

---

## Right-click menu

Right-clicking anywhere on the Workflow Editor canvas gives two options:

| Option | Description |
|---|---|
| Hide Minimap | Toggles the minimap in the bottom right corner |
| Close Workflow | Closes the workflow and returns to the Canvas |

---

## Actions

Actions are the building blocks of every workflow. For a full reference of all available actions see the [Action Directory](/actions/action-directory).

---

## Side navigation

While in the Workflow Editor, the side navigation shows:

- **AI Chat** — the Scram AI assistant
- **Workflow Config** — settings and configuration for the current workflow, including its trigger and authentication settings
