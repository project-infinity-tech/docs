---
title: "Dates and Times"
description: "Storing, querying, and formatting temporal data in ScramDB."
---

Scram represents date and time values as **Temporal objects** from the TC39 Temporal proposal ‚Äî not JavaScript Date objects. There are three types depending on the field:

- `PlainDate` ‚Äî a calendar date with no time
- `PlainTime` ‚Äî a time of day with no date
- `PlainDateTime` ‚Äî a combined date and time

In expressions and component bindings, use Scram's built-in date functions to work with these. When passing dates as SQL parameters, the rules are different ‚Äî see below.

---

## Available field types

| Type | Stores | Example |
|---|---|---|
| `date` | A calendar date only, no time | `2026-04-28` |
| `time` | A time of day only, no date | `14:30:00` |
| `datetime` | A combined date and time | `2026-04-28T14:30:00` |

<Warning>
  `date` and `datetime` are different types ‚Äî don't mix them. A `date` field cannot store a time component, and comparing a `date` field to a full `datetime` value in a query may produce unexpected results.
</Warning>

---

## Storing dates as SQL parameters

The rules for passing date values as named parameters:

| Approach | Works? | Notes |
|---|---|---|
| `new Date()` | ‚úÖ | Use for "right now" |
| Date string with SQL cast | ‚úÖ | e.g. `:myDate::date` or `:myDate::timestamptz` |
| `new Date().toISOString()` | ‚ùå | Serialises to a string PostgreSQL rejects |
| `Now("UTC")` or Temporal objects | ‚ùå | Also rejected as SQL parameters |

**Storing the current moment:**

```js
params: { createdAt: new Date() }
```

**Storing a date string from user input** ‚Äî cast it in the SQL query:

```sql
INSERT INTO "Events" ("date") VALUES (:eventDate::date)
```

<Note>
  The date picker component returns a string, not a date object. Always cast it in SQL rather than relying on automatic type coercion.
</Note>

---

## Querying by date

Use PostgreSQL interval syntax for relative date ranges:

```sql
WHERE "createdAt" > NOW() - INTERVAL '7 days'
```

**Timezone handling** ‚Äî `datetime` fields in ScramDB are stored in UTC. When displaying them, always convert to the user's timezone using Scram's built-in formatting functions rather than doing timezone arithmetic manually in SQL.

## Built-in date functions for expressions

These functions work directly in expressions and component bindings ‚Äî no workflow step needed.

| Function | What it does |
|---|---|
| `Now(timezone)` | Current datetime in a given timezone |
| `FormatDate(date)` | Human-readable date string, e.g. 28 April 2026 |
| `FormatDate(date, true)` | Long form, e.g. Monday, 28 April 2026 |
| `FormatDateTime(date)` | Human-readable date and time, e.g. 28 April 2026, 14:30 |
| `FormatTime(time)` | Human-readable time, e.g. 14:30 |
| `FormatTime(time, true)` | 12-hour clock, e.g. 2:30 PM |
| `FormatTimeBetween(a, b)` | Relative time, e.g. 3 hours ago |
| `ParseDate(string)` | Converts text to a `PlainDate` |
| `ParseDateTime(string, timezone)` | Converts text to a `PlainDateTime` |
| `Add(date, duration)` | Date arithmetic ‚Äî add a duration |
| `Subtract(date, duration)` | Date arithmetic ‚Äî subtract a duration |
| `DaysBetween(a, b)` | Difference in days between two dates |
| `IsToday(date)` | Returns true if the date is today |
| `IsPast(date)` | Returns true if the date is in the past |
| `IsFuture(date)` | Returns true if the date is in the future |
| `Year(date)` | Extracts the year |
| `Month(date)` | Extracts the month |
| `DayOfMonth(date)` | Extracts the day of the month |

<Note>
  These functions are expression-only ‚Äî they work in UI bindings and component expressions. They are not available inside raw SQL queries. Format dates for display on the frontend, not in the database.
</Note>
