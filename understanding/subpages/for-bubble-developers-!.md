# For Bubble Developers !

# **Welcome Bubble Developers !**

Many of the team at Scram have been Bubble users for many years, and of course users of other Visual Development tools, so a lot of the concepts in Scram will be familar to you. We have pages that have visual elements on, we have databases, we have ways to acccess that data.

However, there will be many areas where we have done things slightly differently, so this section of the Scramanual is to highlight those areas.

| Bubble Concept | Scram Concept | Notes |
| --- | --- | --- |
|  | Project | A Scram Project is one or more Scram Apps that share data and components, so not a concept that translates to Bubble. |
| Project | App | A Scram Project contains multiple Apps that share the same data etc |
| Plugins | “Run Code” and “React Components” |  |
| Templates |  |  |
| Pages | Pages | Phew 🙂 |
| - Design Tab |  |  |
| - Workflow Tab |  |  |
| - Repeating Group Data on Hidden Popup | Page Data | If you are familar with having a repeating group on a popup on a page to load data and then filter / sort etc then Page Data is a more formalised way of doing this |
| Custom State | Variable |  |
| Reusable Element | Component |  |
| - Property | - Property |  |
| Type of Content Custom PropertiesCustom States |  | Scram reuses the React term "Props" here, but they are similar
(note that Props behave like Bubble Content Types and Custom Properties rather than Custom States - when the prop's referenced data changes the Component reacts to that)  |
| - Trigger Custom Event | Emit Event | Scram reuses the React term "Emit Event" but they are very similar |
| API Connector Plugin | Data Sources (API) | Scram moves the management of APIs into the Data Sources menu and allows you to import OpenAPI and Postman files directly. |
