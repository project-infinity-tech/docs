Accessibility determines whether everyone can use what you build — including people who navigate with a keyboard, use a screen reader, or rely on other assistive technology. Scram's components are designed to expose the information that assistive technology needs, but the choices you make as a builder also affect the result.

This section documents how Scram's components handle accessibility, where you have control, and any current limitations to be aware of.

## Accessible names

Assistive technology identifies an element by its accessible name — the label a screen reader announces when a user reaches it. Where a component shows visible text, that text usually serves as its accessible name. Where a component has no visible text, such as an icon-only control, it needs an explicit label instead.

The pages in this section describe how each component derives its accessible name and when you need to set one yourself.

## Current limitations

Some accessibility-related behaviours are not yet supported on the platform. These are noted on the relevant pages. If you hit an accessibility requirement that Scram doesn't currently address, contact the team — it helps us prioritise what to build next.
