The Storage section of the Resource View gives you a direct view into your app's File Store — the S3-backed cloud storage for any files your app uploads or manages.

---

## What you'll see

Storage displays a list of files and pseudo-directories. These reflect the structure of your File Store as it currently exists — files uploaded by your app or manually through this view.

<Note>
  Directories in the File Store are virtual — they don't exist as real folders. A file appears to be "in" a directory simply because its filename includes a path prefix, e.g. `avatars/user-123.png`. To organise files into a directory, rename the file to include the desired path prefix.
</Note>

---

## Controls

**Refresh** — reloads the file list to show the current state of your File Store.

**Upload** — upload a file directly from here without going through your app. Useful during development for adding test assets or seeding initial content.

---

<Note>
  For full details on how the File Store works and how to use it in your app, see [The Scram File Store](/understanding/understanding-the-filestore).
</Note>
