---

Server Logs gives you a real-time view of activity on your app's backend server — useful for debugging workflows, monitoring errors, and verifying that server-side operations are running as expected.

---

## What's logged

Server Logs records server-side activity automatically. You can also write your own messages to the log from any workflow using the **Log Message** action, making it easy to trace what's happening during a workflow run.

---

## Finding your backend URL

The Server Logs page is also where you'll find the URL of your app's backend server. You'll need this when configuring incoming API calls or integrating with external services that need to call into your app.

---

<Note>
  Logs are for the Live environment. Check your Dev deployment logs in the Deployments page if you need to debug a Dev build.
</Note>
